# UI Flow Sprint 1 (MVP)

## 1. Generate flow
1. Пользователь открывает экран Generate.
2. Выбирает шаблон и количество строк.
3. Нажимает `Generate`.
4. Frontend вызывает `POST /generate`.
5. При `200` запускается скачивание CSV.
6. При ошибке показывается сообщение из `detail`.

## 2. Anonymize flow
1. Пользователь открывает экран Anonymize.
2. Загружает CSV-файл.
3. При необходимости вызывается `POST /suggest`.
4. Пользователь настраивает методы по колонкам: `keep`, `mask`, `redact`, `pseudo`, `generalize_date`.
5. Нажимает `Anonymize`.
6. Frontend вызывает `POST /anonymize` с `file` и `rules`.
7. При `200` запускается скачивание анонимизированного CSV.
8. При ошибке показывается `detail` и сохраняется состояние формы.

## 3. Правила скачивания
1. Скачивание должно быть явным и повторяемым.
2. Имя файла берется из `Content-Disposition`.
3. При блокировке автоскачивания показывается fallback-кнопка.

## 4. Правила обработки ошибок
1. Пустой CSV — понятное сообщение о пустом файле.
2. Поврежденный CSV/неверный delimiter — сообщение об ошибке парсинга.
3. Неверный метод/валидация — сообщение с проблемным полем.
4. Сетевая ошибка — предложение повторить запрос.

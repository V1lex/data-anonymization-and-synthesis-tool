# UI Flow Sprint 1 (MVP)

## 1. Сценарий Generate
1. Пользователь открывает экран Generate.
2. Выбирает один или несколько шаблонов.
3. Указывает число строк.
4. Нажимает `Generate`.
5. Frontend отправляет `POST /generate`.
6. При `200` запускается скачивание CSV.
7. При ошибке отображается читаемое сообщение из `detail`.

## 2. Сценарий Anonymize
1. Пользователь открывает экран Anonymize.
2. Загружает CSV.
3. Frontend при необходимости вызывает `POST /suggest` для подсказок.
4. UI показывает список колонок и действие по умолчанию `keep`.
5. Пользователь задает методы (`keep/mask/redact/pseudo`).
6. Нажимает `Anonymize`.
7. Frontend отправляет `POST /anonymize` (`file` + `rules`).
8. При `200` запускается скачивание анонимизированного CSV.
9. При ошибке UI выделяет проблемное место и показывает `detail`.

## 3. Правила UX для скачивания
1. Действие скачивания явное и повторяемое.
2. Имя файла берется из `Content-Disposition`.
3. Если браузер блокирует автоскачивание, показывается fallback-кнопка/ссылка.

## 4. Правила UX для ошибок
1. Пустой CSV -> сообщение уровня «загруженный файл пуст».
2. Плохой delimiter/битый CSV -> сообщение с подсказкой про формат.
3. Неправильный метод/валидация -> сообщение о конкретном поле.
4. Сетевая/серверная ошибка -> предложение повторить запрос без потери формы.

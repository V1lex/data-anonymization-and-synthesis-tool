# Тест-план Sprint 1 (MVP)

## 1. Цель
Подготовить основу для smoke/e2e-тестов Sprint 2 по сценариям Generate и Anonymize.

## 2. Область покрытия
1. Генерация CSV.
2. Анонимизация произвольного CSV.
3. Скачивание результата.
4. Негативные проверки входных данных и API.

## 3. Уровни проверок
1. Контрактные (статусы, заголовки, формат запроса/ответа).
2. Функциональные (структура CSV, строки, применение правил).
3. Негативные (невалидные параметры, плохие файлы, неверные методы).
4. Smoke/E2E (сквозные сценарии).

## 4. Критерии приемки

### 4.1 Generate CSV
1. Валидный запрос (`template`, `rows > 0`) возвращает `200`.
2. Ответ имеет `Content-Type: text/csv`.
3. Ответ имеет `Content-Disposition` с `.csv`.
4. CSV-выгрузка соответствует запрошенному шаблону.
5. Количество строк данных равно `rows`.
6. Неизвестный шаблон -> `400/422`.
7. `rows <= 0` -> `400/422`.
8. Некорректный delimiter -> `400/422`.

### 4.2 Anonymize CSV
1. Валидный CSV + валидные `rules` -> `200`.
2. Порядок колонок в выходе совпадает с входом.
3. Количество строк в выходе совпадает с входом.
4. `keep` не меняет значение.
5. `mask` меняет значение на маскированный вид.
6. `redact` скрывает значение.
7. `pseudo` дает консистентный результат для одинаковых входов в одной колонке.
8. `generalize_date` обобщает дату по контракту метода.
9. Неизвестная колонка в `rules` -> `400/422`.
10. Неподдерживаемый `method` -> `400/422`.

### 4.3 Скачивание результата
1. После `POST /generate` файл доступен к скачиванию.
2. После `POST /anonymize` файл доступен к скачиванию.
3. Скачанный файл корректно открывается как CSV.
4. Имя файла в `Content-Disposition` непустое.

## 5. Обязательные негативные кейсы
1. Пустой CSV -> `400/422`.
2. Поврежденный CSV -> `400/422`.
3. Плохой delimiter -> предсказуемая ошибка разбора или корректный autodetect (если включен).
4. Не-CSV файл с расширением `.csv` -> `400/422` при ошибке парсинга.
5. Неправильный HTTP-метод -> `405`.
6. Отсутствие обязательных полей -> `422`.
7. Неверные типы полей (`rows` строкой, `rules` массивом) -> `422`.

## 6. Smoke/E2E базовый набор
1. `GET /health` -> `200`.
2. Generate: запрос -> скачивание -> проверка структуры и количества строк.
3. Anonymize: загрузка CSV + правила -> скачивание -> проверка преобразований.
4. Негативный smoke: неправильный HTTP-метод -> `405`.
5. Негативный smoke: пустой CSV -> ожидаемая ошибка.

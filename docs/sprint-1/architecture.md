# Архитектура репозитория

## Цель
Единообразно описать слои и их ответственность для Sprint 1.

## Слои
1. `src/sda/core/` — доменная логика (генерация, анонимизация, эвристики).
2. `src/sda/io/` — чтение/запись CSV.
3. `src/sda/use_cases/` — прикладные сценарии, связывающие домен и ввод/вывод.
4. `src/sda/web/routers/` — HTTP-эндпоинты.
5. `src/sda/web/schemas/` — Pydantic-схемы API.
6. `src/sda/resources/templates/` — шаблоны генерации.

## Правила зависимостей
1. `core` не зависит от FastAPI и HTTP-типов.
2. `web` не реализует бизнес-логику напрямую, а вызывает `use_cases`.
3. `io` не содержит доменных решений.
4. `schemas` описывают контракты API, а не доменные сущности.

## Антипаттерны
1. Дублирование логики между `routers` и `use_cases`.
2. Чтение/запись CSV напрямую в роутерах.
3. Привязка доменного слоя к HTTP-кодам.

# Методы анонимизации (MVP)

## 1. Назначение
Настоящий документ определяет перечень и логику работы методов анонимизации, поддерживаемых в MVP. Система спроектирована для обработки произвольных CSV-файлов, где выбор метода осуществляется пользователем индивидуально для каждой колонки.

**Ключевые требования:**
- Независимость от структуры входных данных.
- Возможность применения методов к любым текстовым и датоподобным полям.
- Сохранение типов данных там, где это критично для бизнес-логики.

---

## 2. Метод `keep` (Без изменений)
Возвращает исходное значение без модификаций.
- **Применение**: Нечувствительные данные (города, категории, статусы).
- **Пример**: `Moscow` -> `Moscow`.
- **Особенности**: Поддерживает любые типы данных.

---

## 3. Метод `pseudonymize` (Псевдонимизация)
Заменяет исходное значение на стабильный детерминированный идентификатор.
- **Логика**: Одинаковые входные значения преобразуются в идентичные выходные в рамках одного набора данных.
- **Применение**: ФИО, Email, ID пользователей (сохранение связности записей).
- **Пример**: `ivanov@mail.ru` -> `ps_4f8a2c91`.
- **Особенности**: Необратимый метод; требует использования криптографической соли.

---

## 4. Метод `mask` (Маскирование)
Частично скрывает содержимое строки, сохраняя её читаемость и формат.
- **Логика**: Замена центральной части строки на маску (символы `*`).
- **Применение**: Номера телефонов, Email, номера карт.
- **Пример**: `+79991234567` -> `+7******67`.
- **Особенности**: Только для строковых полей.

---

## 5. Метод `redact` (Полное скрытие)
Заменяет значение на фиксированный маркер.
- **Логика**: Полное удаление оригинальной информации.
- **Применение**: Адреса, свободные комментарии, паспортные данные.
- **Пример**: `ул. Ленина, 10` -> `[REDACTED]`.
- **Особенности**: По умолчанию используется значение `[REDACTED]`.

---

## 6. Метод `generalize_year` (Обобщение даты)
Снижает точность временных данных до уровня года.
- **Логика**: Извлечение четырехзначного года из строкового представления даты.
- **Применение**: Даты рождения, даты регистрации событий.
- **Пример**: `1990-05-15` -> `1990`.
- **Особенности**: Предназначен исключительно для полей типа «Дата».

---

## 7. Общие правила обработки
1. **Пустые значения**: Если ячейка содержит `null` или пустую строку, метод возвращает пустое значение без обработки.
2. **Валидация**: Попытка применить метод к несовместимому типу данных (например, `generalize_year` к текстовому полю) должна приводить к ошибке валидации на уровне бизнес-логики.
3. **Расширяемость**: Архитектура позволяет добавлять новые методы через расширение `AnonymizationMethod` и `METHOD_SPECS`.

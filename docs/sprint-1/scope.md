# Scope Sprint 1: границы MVP

## Цель
Зафиксировать, что входит в MVP для Показа 1, а что переносится на следующий этап.

## Входит в MVP
1. Ветка `Generate`.
2. Ветка `Anonymize`.

## Не входит в MVP (после Показа 1)
1. Полноценная ветка `Similar`.
2. Расширенные метрики качества synthetic-данных.
3. Нагрузочное и security-тестирование.

## Сценарий `Generate`
1. Пользователь выбирает шаблон.
2. Указывает количество строк.
3. Получает CSV для скачивания.

## Сценарий `Anonymize`
1. Пользователь загружает произвольный CSV.
2. Настраивает правила по колонкам.
3. Получает анонимизированный CSV для скачивания.

## Поддерживаемые методы анонимизации в MVP
1. `keep`
2. `mask`
3. `redact`
4. `pseudo`
5. `generalize_date`
